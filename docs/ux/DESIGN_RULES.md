# Tavyn UX Design Rules

- Clarity over decoration.
- Founder control over automation magic.
- Trust before GitHub access.
- Preview before commitment.
- No blind publishing.
- Show what Tavyn found.
- Explain what Tavyn will do next.
- Make the GitHub PR workflow feel safe and premium.
- Avoid bulk-content-tool aesthetics.

## Product Feel

Tavyn should feel premium, calm, technical, trustworthy, founder-controlled, minimal, and GitHub-native.

The reference set is closer to Linear, Vercel, GitHub, Superhuman, and Resend than Jasper or Copy.ai.

## Visual Direction

- Mostly white background.
- Black primary text.
- Soft borders.
- Strong whitespace.
- Subtle orange-red gradient accents used sparingly.
- Clean typography.
- Minimal cards.
- No stock illustrations.
- No AI robot imagery.
- No loud gradients.
- No generic AI dashboard clutter.
